Docker est la méthode de déploiement **recommandée** pour Worcable. Elle garantit une isolation parfaite des services et une reproductibilité totale, que ce soit sur votre machine locale ou sur un serveur de production.

L'écosystème Docker de Worcable orchestre trois composants clés par défaut :
- **L'Application Nuxt** (Fullstack : UI + API)
- **La Base de données** (MongoDB)
- **Les Services Indépendants** (ex: CV Parser)

## Prérequis

Assurez-vous que votre système dispose des outils suivants :

* **Docker** ≥ 20.10
* **Docker Compose** ≥ 2.0

Vérifiez votre installation avec ces commandes :
```bash [Terminal]
docker -v
docker compose version
```

## Installation et Configuration

### 1\. Récupérer le projet

```bash [Terminal]
git clone [https://github.com/domutala/worcable.git](https://github.com/domutala/worcable.git)
cd worcable
```

### 2\. Variables d'environnement

Copiez le fichier d'exemple à la racine du projet et ajustez les ports si nécessaire :

```bash [Terminal]
cp .env.example .env
```

::callout{icon="i-lucide-info" color="info"}
Par défaut, Worcable utilise le port **3000** pour l'interface, **27017** pour MongoDB et **5000** pour le service de parsing.
::

## Utilisation du Script d'Orchestration

Worcable inclut un script utilitaire situé dans `./deploy/run.sh` pour simplifier la gestion des conteneurs.

### Rendre le script exécutable

```bash [Terminal]
chmod +x deploy/run.sh
```

### Lancement des services

Le script vous permet d'activer les modules spécifiques dont vous avez besoin en les listant simplement comme arguments.

::callout{icon="i-lucide-layers" color="primary"}
**Note sur les services indépendants** : La liste des services est modulaire. Pour activer un service spécifique (comme le `cvparser`), il suffit de passer son nom au script.
::

| Mode | Commande | Description |
| :--- | :--- | :--- |
| **Full Stack** | `./deploy/run.sh all` | **Par défaut** : Lance Nuxt, MongoDB et tous les services actifs. |
| **Mode Base de données** | `./deploy/run.sh base db` | Lance l'App Nuxt et la base MongoDB uniquement. |
| **Service Spécifique** | `./deploy/run.sh base cvparser` | Lance l'App Nuxt et le service indépendant ciblé. |
| **App Uniquement** | `./deploy/run.sh base` | Lance uniquement le conteneur Nuxt. |

## Accès aux Services

Une fois les conteneurs démarrés, les services sont accessibles aux adresses suivantes :

| Service | URL / Point d'accès |
| :--- | :--- |
| **Interface & API** | [http://localhost:3000](https://www.google.com/search?q=http://localhost:3000) |
| **Base de données** | `mongodb://localhost:27017` |
| **CV Parser API** | [http://localhost:5000](https://www.google.com/search?q=http://localhost:5000) |

## Maintenance et Logs

### Consulter les journaux (Logs)

Pour suivre les logs de vos services actifs :

```bash [Terminal]
./deploy/run.sh all --logs
```

### Arrêter les services

Pour stopper proprement tous les conteneurs liés au projet :

```bash [Terminal]
./deploy/run.sh all --stop
```

### Mise à jour et Reconstruction

Si vous modifiez le code source (notamment dans `package/` ou un service indépendant), forcez la reconstruction des images :

```bash [Terminal]
docker compose -f deploy/docker/docker-compose.yml up --build -d
```

## Recommandations pour la Production

::callout{icon="i-lucide-shield-check" color="warning"}
Pour un déploiement exposé sur internet, suivez ces règles de sécurité :
::

1.  **Reverse Proxy** : Utilisez Nginx ou Traefik pour gérer le SSL (HTTPS).
2.  **Isolation** : Ne mappez pas le port 27017 de MongoDB sur l'hôte.
3.  **Secrets** : Changez impérativement toutes les clés secrètes et mots de passe dans votre fichier `.env`.
