---
title: Déploiement Natif 
description: Guide pour installer et lancer Worcable directement sur votre système ou serveur VPS.
---

Le mode de déploiement natif est idéal pour les environnements où Docker n'est pas disponible, pour le développement local intensif ou pour optimiser l'utilisation des ressources sur des serveurs légers.

Dans ce mode, vous gérez vous-même les processus via le script d'orchestration qui s'appuie sur **Node.js**, **Python** et une instance **MongoDB** locale ou distante.

## Prérequis

Avant de commencer, assurez-vous que les éléments suivants sont installés sur votre machine :

* **Node.js** (v18.x ou supérieur)
* **pnpm** (Recommandé pour la gestion du monorepo)
* **Python** (v3.9 ou supérieur + `pip` et `venv`)
* **MongoDB** (Instance locale ou URI distante accessible)

Vérifiez vos versions :
```bash [bash]
node -v
pnpm -v
python3 --version
```

## Installation et Configuration

### 1. Cloner le dépôt
```bash [bash]
git clone [https://github.com/domutala/worcable.git](https://github.com/domutala/worcable.git)
cd worcable
```

### 2. Configuration des variables d'environnement
Chaque module (App Nuxt et CV Parser) nécessite sa propre configuration. Copiez les fichiers `.env` à la racine :

```bash [bash]
cp .env.example .env
```

::callout{icon="i-lucide-database" color="warning"}
**Important** : En mode natif, assurez-vous que votre variable `DATABASE_URL` dans le `.env` pointe vers une instance MongoDB active (ex: `mongodb://localhost:27017/worcable`).
::

## Utilisation du Script d'Orchestration

Worcable fournit un script unique pour gérer le lancement de tous les services en mode natif. Il se charge de créer les environnements virtuels Python et d'installer les dépendances Node.js automatiquement.

### Rendre le script exécutable
```bash [bash]
chmod +x deploy/run.sh
```

### Lancement avec le flag `--native`
Pour utiliser le mode natif, vous devez impérativement ajouter le flag `--native` à vos commandes.

| Commande | Description |
| :--- | :--- |
| `./deploy/run.sh all --native` | Lance l'App Nuxt et le CV Parser en simultané. |
| `./deploy/run.sh base --native` | Lance uniquement l'application principale Nuxt. |
| `./deploy/run.sh cvparser --native` | Lance uniquement le service de parsing Python. |

## Gestion des Services

### Arrêt des services
Contrairement à Docker qui tourne en arrière-plan, le mode natif lie les processus à votre terminal.
* Pour arrêter tous les services, utilisez simplement **`CTRL+C`**.
* Le script interceptera le signal et fermera proprement les serveurs Nuxt et Python.

### Logs
Les journaux d'exécution s'affichent directement dans votre terminal. Pour une mise en production, il est recommandé d'utiliser un gestionnaire de processus comme **PM2**.

## Recommandations pour la Production

::callout{icon="i-lucide-alert-triangle" color="primary"}
Si vous déployez Worcable en mode natif sur un serveur de production (VPS) :
::

1.  **Gestionnaire de processus** : Utilisez **PM2** pour garantir que vos services redémarrent automatiquement après un crash ou un reboot du serveur.
2.  **Reverse Proxy** : Configurez **Nginx** ou **Apache** pour rediriger le trafic du port 80/443 vers le port 3000 de Nuxt.
3.  **Pare-feu** : Ne laissez pas le port 27017 (MongoDB) ouvert vers l'extérieur sans authentification stricte.
