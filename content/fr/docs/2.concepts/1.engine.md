L’architecture de Worcable est conçue pour être **modulaire, évolutive et orientée monorepo**, afin de faciliter le développement, le déploiement et la maintenance de chaque composant.

Elle repose sur une séparation claire entre le cœur de l'application, les services spécialisés et l’infrastructure de déploiement.

## Architecture Globale

Worcable est organisé sous forme de **monorepo structuré** permettant une gestion unifiée du code source:

```text
.
├── package/      # Application principale (Nuxt 3 fullstack)
├── cvparser/     # Service indépendant de parsing de CV (Python)
├── deploy/       # Infrastructure et orchestrateur Docker
└── docs/         # Documentation technique (Nuxt Content)
```

## Applications du Monorepo

Worcable distingue le noyau applicatif des **Services Indépendants**, permettant d'isoler les traitements spécifiques.

| Module        | Rôle | Détails | Objectif |
|--------------|------|--------|----------|
| `package/` | Core Application | - UI (pages, composants)<br>- API backend (`/server/api`)<br>- Logique métier<br>- Accès base de données | Cœur du produit |
| `docs/` | Documentation | - Basée sur **Nuxt Content**<br>- Documentation versionnée<br>- Support i18n | DX propre et scalable |
| `cvparser/` | Service de parsing | - Parsing PDF / DOCX<br>- Extraction de données<br>- Structuration des profils candidats<br><br>**Stack :**<br>- Python<br>- FastAPI<br>- NLP (spaCy, etc.) | Performance, scalabilité, évolution IA |
| `deploy/` | Infrastructure | - `docker-compose.yml`<br>- Dockerfiles (base + services)<br>- Scripts de lancement | Déploiement reproductible |

## Scalabilité

Worcable repose sur une architecture **monolithique modulaire**. Le frontend et le backend coexistent dans `package/` tout en respectant une séparation stricte des responsabilités. Ce noyau délègue les tâches intensives à des **services spécialisés** (comme `cvparser`), permettant d’externaliser les calculs lourds.

L'ensemble est orchestré via **Docker**, garantissant un déploiement cohérent sur n'importe quel environnement, du VPS au Cloud. Cette structure permet d'évoluer vers un système distribué : il est possible d'introduire des files de messages pour les traitements asynchrones ou d'extraire des briques logiques vers des frameworks dédiés si la charge l'exige

## Deployment

Worcable offers total flexibility for production, adapting to your infrastructure constraints through a unified orchestrator:

### Option 1: Docker (Recommended)
Full orchestration via Docker Compose. This is the preferred method for isolation and easy scaling.
```bash [Terminal]
# Launch everything (Nuxt, DB, Parser)
./deploy/run.sh all

# Launch specific services (e.g., Nuxt + MongoDB)
./deploy/run.sh base db

# Launch and follow logs for a specific service
./deploy/run.sh cvparser --logs
```

::callout{icon="i-simple-icons-docker" color="info" to="/en/docs/deployment/docker"}
Read the complete guide to deploy Worcable with Docker in production environments.
::

### Option 2: Native (Shell Script)
For environments without Docker. Requires **Node.js (pnpm)**, **Python 3**, and **MongoDB** installed locally.
```bash [Terminal]
# Launch everything natively
./deploy/run.sh all --native

# Launch only the Nuxt application
./deploy/run.sh base --native

# Launch specific services natively
./deploy/run.sh base cvparser --native
```

::callout{icon="i-lucide-app-window" color="info" to="/en/docs/deployment/native"}
Read the complete guide to deploy Worcable in native mode.
::

## Philosophie

Worcable est conçu pour :

  * **Rester simple** à comprendre pour les contributeurs open source.
  * **Être rapide** à déployer sans configuration complexe.
  * **Garantir la souveraineté** des données via l'auto-hébergement.
  * **S'adapter** aux environnements à ressources limitées.
