:icon{name="i-lucide-chevrons-right"} Une seule commande suffit pour installer et lancer votre infrastructure.

---

## Installation du CLI

Avant tout, installez le CLI Worcable sur votre machine :

```bash [Terminal]
curl -o- https://raw.githubusercontent.com/domutala/worcable/install.sh | bash
```

::callout{icon="i-lucide-lightbulb" color="success" }
ette commande installe l’outil `worcable` globalement sur votre système.
::

## Lancer un déploiement

Une fois le CLI installé :

```bash [Terminal]
worcable deploy
```

:icon{name="i-lucide-chevrons-right"} Le CLI démarre un assistant interactif qui vous guide étape par étape.

---

## Configuration interactive

Pendant le déploiement, Worcable vous pose quelques questions essentielles :

### Informations projet

* Votre nom
* Votre email
* Nom de l’organisation

### Configuration réseau

* Domaine (ex: `career.your-domain.com`)
* Protocole (HTTP / HTTPS)

### Infrastructure

* Méthode de déploiement (Docker recommandé)
* Réseau Docker

### Services à installer

* Core (API + application)
* Database (MongoDB)
* ...

### Configuration base de données

* Port
* Username
* Mot de passe (auto-généré si vide)

### Configuration application

* Port
* URL publique
* Clé secrète (auto-générée si vide)
* Connexion à la base de données (générée automatiquement)

## Ce que fait Worcable automatiquement

Une fois la configuration terminée, le CLI s’occupe de tout :

* Génération des fichiers de configuration
* Création des secrets sécurisés
* Configuration du réseau Docker
* Déploiement des services
* Connexion automatique entre les services

:icon{name="i-lucide-chevrons-right"} Aucun fichier `.env` à éditer
<br /> :icon{name="i-lucide-chevrons-right"} Aucune configuration Docker à écrire

## Fichiers générés

Worcable stocke la configuration localement :

```bash
~/.worcable/<version>
  ├── config.json
  └── docker.json
```

:icon{name="i-lucide-chevrons-right"} Ces fichiers permettent de :

* Rejouer un déploiement
* Mettre à jour votre installation
* Gérer votre infrastructure

## Une architecture modulaire

Worcable conserve une approche **modulaire et extensible**.

Lors du déploiement, vous choisissez les composants à installer :

* **Core** : API et application principale
* **Database** : MongoDB
* services supplémentaires :

  * CVTheque
  * Notifications
  * IA / scoring

:icon{name="i-lucide-chevrons-right"} Vous pouvez adapter votre stack selon vos besoins.


## Infrastructure générée

Le CLI orchestre automatiquement :

* Containers Docker
* Réseau interne
* Variables d’environnement
* Connexions entre services

:icon{name="i-lucide-chevrons-right"} Vous obtenez une infrastructure prête à l’emploi, sans manipulation manuelle.

Worcable adopte une approche :  **“Deployment as Code”**

Si vous rencontrez un problème :

* Relancez simplement `worcable deploy`
* Vérifiez vos paramètres réseau (domaine, ports)
