Docker is the **recommended** deployment method for Worcable. It ensures perfect service isolation and total reproducibility, whether on your local machine or a production server.

The Worcable Docker ecosystem orchestrates three key components by default:
* **The Nuxt Application** (Fullstack: UI + API)
* **The Database** (MongoDB)
* **Independent Services** (e.g., CV Parser)

## Prerequisites

Ensure your system has the following tools installed:

* **Docker** ≥ 20.10
* **Docker Compose** ≥ 2.0

Verify your installation with these commands:
```bash [Terminal]
docker -v
docker compose version
````

## Installation and Configuration

### 1\. Clone the Project

```bash [Terminal]
git clone https://github.com/domutala/worcable.git
cd worcable
```

### 2\. Environment Variables

Copy the example file to the project root and adjust ports if necessary:

```bash [Terminal]
cp .env.example .env
```

::callout{icon="i-lucide-info" color="info"}
By default, Worcable uses port **3000** for the interface, **27017** for MongoDB, and **5000** for the parsing service.
::

## Using the Orchestration Script

Worcable includes a utility script located at `./deploy/run.sh` to simplify container management.

### Make the script executable

```bash [Terminal]
chmod +x deploy/run.sh
```

### Launching Services

The script allows you to enable specific modules by listing them as arguments.

::callout{icon="i-lucide-layers" color="primary"}
**Note on independent services**: The service list is modular. To activate a specific service (like `cvparser`), simply pass its name to the script.
::

| Mode | Command | Description |
| :--- | :--- | :--- |
| **Full Stack** | `./deploy/run.sh all` | **Default**: Launches Nuxt, MongoDB, and all active services. |
| **Database Mode** | `./deploy/run.sh base db` | Launches the Nuxt App and the MongoDB database only. |
| **Specific Service** | `./deploy/run.sh base cvparser` | Launches the Nuxt App and the targeted independent service. |
| **App Only** | `./deploy/run.sh base` | Launches only the Nuxt container. |

## Accessing Services

Once the containers are started, services are accessible at the following addresses:

| Service | URL / Access Point |
| :--- | :--- |
| **Interface & API** | [http://localhost:3000](https://www.google.com/search?q=http://localhost:3000) |
| **Database** | `mongodb://localhost:27017` |
| **CV Parser API** | [http://localhost:5000](https://www.google.com/search?q=http://localhost:5000) |

## Maintenance and Logs

### View Logs

To follow the logs of your active services:

```bash [Terminal]
./deploy/run.sh all --logs
```

### Stop Services

To properly stop all containers related to the project:

```bash [Terminal]
./deploy/run.sh all --stop
```

### Update and Rebuild

If you modify the source code (especially in `package/` or an independent service), force a rebuild of the images:

```bash [Terminal]
docker compose -f deploy/docker/docker-compose.yml up --build -d
```

## Production Recommendations

::callout{icon="i-lucide-shield-check" color="warning"}
For a deployment exposed to the internet, follow these security rules:
::

1.  **Reverse Proxy**: Use Nginx or Traefik to manage SSL (HTTPS).
2.  **Isolation**: Do not map MongoDB port 27017 to the host.
3.  **Secrets**: It is imperative to change all secret keys and passwords in your `.env` file.

::
