title: Native Deployment
description: Guide to installing and launching Worcable directly on your system or a VPS server.
---

Native deployment is ideal for environments where Docker is not available, for intensive local development, or for optimizing resource usage on lightweight servers.

In this mode, you manage the processes yourself via the orchestration script, which relies on **Node.js**, **Python**, and a local or remote **MongoDB** instance.

## Prerequisites

Before starting, ensure that the following elements are installed on your machine:

* **Node.js** (v18.x or higher)
* **pnpm** (Recommended for monorepo management)
* **Python** (v3.9 or higher + `pip` and `venv`)
* **MongoDB** (Local instance or accessible remote URI)

Verify your versions:
```bash [bash]
node -v
pnpm -v
python3 --version
```

## Installation and Configuration

### 1. Clone the repository
```bash [bash]
git clone https://github.com/domutala/worcable.git
cd worcable
```

### 2. Environment Variables Configuration
Each module (Nuxt App and CV Parser) requires its own configuration. Copy the `.env` files to the root:

```bash [bash]
cp .env.example .env
```

::callout{icon="i-lucide-database" color="warning"}
**Important**: In native mode, ensure that your `DATABASE_URL` variable in the `.env` file points to an active MongoDB instance (e.g., `mongodb://localhost:27017/worcable`).
::

## Using the Orchestration Script

Worcable provides a single script to manage the launch of all services in native mode. It handles the creation of Python virtual environments and the installation of Node.js dependencies automatically.

### Make the script executable
```bash [bash]
chmod +x deploy/run.sh
```

### Launching with the `--native` flag
To use native mode, you must include the `--native` flag in your commands.

| Command | Description |
| :--- | :--- |
| `./deploy/run.sh all --native` | Launches the Nuxt App and CV Parser simultaneously. |
| `./deploy/run.sh base --native` | Launches only the main Nuxt application. |
| `./deploy/run.sh cvparser --native` | Launches only the Python parsing service. |

## Service Management

### Stopping services
Unlike Docker, which runs in the background, native mode binds processes to your terminal.
* To stop all services, simply use **`CTRL+C`**.
* The script will intercept the signal and cleanly shut down the Nuxt and Python servers.

### Logs
Execution logs are displayed directly in your terminal. For production environments, it is recommended to use a process manager like **PM2**.

## Production Recommendations

::callout{icon="i-lucide-alert-triangle" color="primary"}
If you are deploying Worcable in native mode on a production server (VPS):
::

1.  **Process Manager**: Use **PM2** to ensure your services restart automatically after a crash or server reboot.
2.  **Reverse Proxy**: Configure **Nginx** or **Apache** to redirect traffic from port 80/443 to Nuxt's port 3000.
3.  **Firewall**: Do not leave port 27017 (MongoDB) open to the outside without strict authentication.

::callout{icon="i-simple-icons-docker" color="info" to="/en/docs/deployment/docker"}
Do you prefer container isolation? Consult our Docker deployment guide.
::
